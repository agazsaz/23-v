# Конспект: Основы компьютерных сетей

## 1. Основные понятия

### Сеть (Network)
Взаимодействующая совокупность объектов (**узлов**, nodes).

### Компьютерная сеть (Computer Network)
Совокупность связанных между собой:
- Компьютеров
- Телекоммуникационного оборудования  
- Программного обеспечения

**Назначение:** обеспечение информационного обмена между компьютерами.

### Ключевые элементы:
- **Узел (хост)** – конечная система
- **Линии связи** – соединяют узлы
- **Коммутаторы пакетов** – обеспечивают передачу данных
- **Пакеты** – отдельные порции передаваемой информации

## 2. Состав компьютерной сети
1. **Компьютеры** (соответствующие назначению сети)
2. **Коммуникационное оборудование**
3. **Сетевые операционные системы** (NOS – Network Operation System)
4. **Сетевые приложения**

## 3. Телекоммуникации и сети

### Телекоммуникации
Передача и прием информации (звук, изображение, данные, текст) на расстояние по электромагнитным системам.

### Телекоммуникационная сеть
Система технических средств для осуществления телекоммуникаций.

**Структура телекоммуникационной сети:**

### Типы телекоммуникационных сетей:
- Компьютерные сети (передача данных)
- Телефонные сети (голосовая информация)
- Радиосети (широковещательные услуги)
- Телеграфные сети (текстовые сообщения)
- Телевизионные сети

## 4. Классификации сетей

### По территориальному признаку:
| Тип сети | Аббревиатура | Описание | Пример |
|----------|--------------|----------|---------|
| Локальная сеть | LAN (Local Area Network) | В пределах помещения, здания или небольшой территории | Сеть университета, офиса |
| Региональная сеть | MAN (Metropolitan Area Network) | Для обслуживания района, города или региона | Городская сеть провайдера |
| Глобальная сеть | WAN (Wide Area Network) | Объединяет территориально рассредоточенные компьютеры | Интернет, сеть Fido |

### По масштабу производственного подразделения:
1. Сети отделов (рабочих групп)
2. Сети кампусов (университетские, домовые)
3. Корпоративные сети (масштаба предприятия)

### По технологии передачи данных:
- **Вещание** (один-ко-многим, broadcast)
- **Точка-точка** (point-to-point)

### По принципу обмена данными:
Сети на основе коммутации:
- Каналов
- Пакетов  
- Сообщений

**Коммутация** – технология выбора направления и организации передачи данных в сетях с несколькими альтернативными маршрутами.

### По типу среды передачи:
- **Проводные** (wired): коаксиальный кабель, витая пара, оптоволокно
- **Беспроводные** (wireless): радиочастоты, инфракрасный диапазон

### По организации иерархии компьютеров:
- **Одноранговые** (Peer-to-Peer)
- **Клиент-серверные** (с выделенным сервером)

**Сервер** – компьютер или программа, предоставляющая услуги.  
**Клиент** – компьютер или программа, запрашивающая услуги.

## 5. Основное назначение компьютерных сетей

### 1. Обеспечение доступа к разделяемым ресурсам
**Разделяемый ресурс** – устройство или информация, доступная удаленно.

**Услуги доступа:**
- Удаленный доступ (Remote Login)
- Передача файлов (File Transfer)
- Удаленный вызов процедур (RPC)
- Совместное использование устройств

### 2. Межперсональная коммуникация
- Электронная почта (1:1)
- Списки рассылки (1:n)
- Телеконференции (n:n)
- Системы электронных бюллетеней (BBS)
- Видеоконференции

## 6. История развития сетей

### Хронология ключевых событий:
| Год | Событие | Значение |
|-----|---------|----------|
| 1960-е | Проект DARPA | Первые эксперименты с объединением компьютеров |
| 1969 | Создание ARPANET | Первая сеть, объединившая суперкомпьютеры США |
| 1971 | Первая программа email | Рей Томлинсон (BBN) |
| 1973 | Изобретение Ethernet | Роберт Меткалф (Xerox) |
| 1974 | Разработка TCP | Винтон Серф и Роберт Кан |
| 1978-79 | Модель OSI | Эталонная модель взаимодействия |
| 1980-е | Стандартизация LAN | Ethernet, Token Ring, FDDI |
| 1988 | Червь Морриса | Первая крупная кибератака, создание CERT |
| 1991-92 | Изобретение WWW | Тим Бернерс-Ли (CERN) |
| 1995 | Коммерциализация Интернета | Появление ISP, современная структура |

## 7. Сетевые модели и протоколы

### Протокол
Набор правил и соглашений, определяющих формат и порядок обмена данными между сетевыми устройствами.

### Стек протоколов (Protocol Stack)
Иерархически организованный набор протоколов, обеспечивающих сетевое взаимодействие.

**Пример структуры стека протоколов:**

### Ключевые протоколы TCP/IP:

| Уровень | Протоколы | Назначение |
|---------|-----------|------------|
| Приложения | HTTP, HTTPS, FTP, SMTP, DNS, DHCP | Взаимодействие приложений |
| Транспортный | TCP, UDP | Доставка данных между приложениями |
| Интернет | IP, ICMP, ARP, IGMP | Маршрутизация и адресация |
| Сетевой доступ | Ethernet, Wi-Fi, PPP | Передача данных по физической среде |

### Принципы работы протоколов:
1. **Инкапсуляция** – добавление заголовков на каждом уровне
2. **Модульность** – независимость уровней
3. **Стандартизация** – единые правила для всех устройств
4. **Совместимость** – возможность взаимодействия разных систем

**Важность сетевых моделей:**
- Обеспечивают совместимость оборудования разных производителей
- Позволяют разделить сложные задачи на более простые
- Упрощают разработку сетевых приложений
- Обеспечивают стандартизацию в индустрии
